<threat_model>
- **T-03-02-01 Duplicate ownership (high):** The playbook could accidentally tell AI Tools to implement kit-owned adoption/bootstrap behavior.
  - **Mitigation:** Add an explicit ownership table and rejection criteria for kit-owned infrastructure.
  - **Verification:** Plan 03-03 docs validation checks playbook language for repo ownership boundaries and non-goals.
- **T-03-02-02 Endless request exchange (medium):** Repos can bounce requests indefinitely without decisions.
  - **Mitigation:** Require decision artifacts, review/expiry fields, non-goals, and stale/superseded statuses.
  - **Verification:** Templates and playbook include expiry/review and decision statuses.
- **T-03-02-03 Hidden automation (high):** Future assistants might treat requests as permission to auto-run tools or modify roadmaps.
  - **Mitigation:** Repeat the no automatic integration, no automatic phase creation, no automatic tool runs, no dependency, and no copied `.planning` boundaries in the playbook and AGENTS.
  - **Verification:** Docs validation checks for the boundary phrases.
</threat_model>
