- D-13: Every registry entry defines id, name, stages, required artifacts, required fields or observable output, skip allowance, and skip reason behavior.
- D-14: Major GSD artifacts created after this phase include `Gate Resolution` when a registered gate applies.
- D-15: Inapplicable or skipped gates require explicit reasons; non-skippable gates cannot be silently skipped.
- D-16: Interactive question fallback requires plain-text choices and stop-for-user-input behavior.
- D-17: Mechanical gate checks are evidence only; semantic gate decisions remain assistant-owned.
</must_haves>

<threat_model>
- **T-03-01-01 Prose-only gate enforcement (high):** Future agents may skip gates again if the rule only exists in prose.
  - **Mitigation:** Create `.planning/gates/registry.json` with stable IDs, stages, required artifacts, observable outputs, and skip rules.
  - **Verification:** Plan 03-03 docs validation parses the registry and checks required gate IDs and fields.
- **T-03-01-02 Invalid request vocabulary (medium):** Requests can drift if boundary classifications or statuses are freeform.
  - **Mitigation:** Put allowed classifications and statuses directly in templates.
  - **Verification:** Plan 03-03 docs validation checks template vocabulary.
- **T-03-01-03 Hidden automation boundary creep (high):** Templates could imply that requests trigger automatic work.
  - **Mitigation:** Include explicit non-goal and decision-needed sections in templates.
  - **Verification:** Playbook and validation in later plans check the no-automatic-phase/tool/dependency language.
</threat_model>
