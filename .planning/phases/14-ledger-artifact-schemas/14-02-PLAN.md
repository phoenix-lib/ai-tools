# Plan 14-02: Generated Ledger Validation, Docs, Registry, and Self-Use Evidence

## Gate Resolution

- **Research Gate:** user selected research-first; evidence is in
  `14-RESEARCH.md`.
- **Dependency Gate:** depends on `14-01` schema files and schema contract
  tests. Do not loosen schemas to pass current output; fix narrow generator
  mismatches instead.
- **AI Tools Self-Use Gate:** applies. After generated output validates,
  produce an external `project-context-ledger` packet and record evidence in
  the summary.
- **Project Changelog Gate:** applies. Update `CHANGELOG.md` with schema
  contract scope, validation, compatibility, and upstream impact.
- **Cross-Repo Gates:** skipped with reason. This plan does not mutate
  `ai-workspace-kit` or make ledger schemas a runtime dependency there.

<objective>
Wire the new ledger artifact schemas into generated-output validation, make the
minimal generator fixes needed for strict schema validity, update docs and
registry evidence, and record self-use evidence from a real generated ledger
packet without changing the ledger into a scope/diff, disposition, or triage
system.
</objective>

<threat_model>
- **T-14-02-01 Schema bypass (high):** generated-output tests could compile
  schemas but not validate actual ledger artifacts.
  - **Mitigation:** validate each generated ledger artifact from the mature
    fixture against its matching schema.
  - **Verification:** test fails when any generated artifact is removed,
    malformed, or not schema-valid.
- **T-14-02-02 Duplicate stable IDs (high):** context decision IDs can collide
  across phase files.
  - **Mitigation:** include deterministic source-path slug or equivalent
    namespace in generated context decision IDs.
  - **Verification:** unique-ID assertion per record artifact.
- **T-14-02-03 Broken evidence refs (high):** ledger records may cite evidence
  IDs that do not exist in `EVIDENCE.json`.
  - **Mitigation:** add cross-artifact join assertions in generated-output
    tests.
  - **Verification:** test fails on a missing evidence id.
- **T-14-02-04 Over-broad generator changes (medium):** schema work could
  accidentally implement Phase 16 scope/diff or Phase 15 dispositions.
  - **Mitigation:** only add schema-version metadata, source-path backfills,
    and stable IDs needed for schema validity.
  - **Verification:** no new CLI options or disposition/scope fields are
    introduced.
- **T-14-02-05 Release/documentation drift (medium):** registry and docs could
  claim validated schemas without evidence.
  - **Mitigation:** cite focused tests, full suite, and self-use output in
    summary/changelog/registry notes.
  - **Verification:** registry/release docs tests plus manual summary review.
</threat_model>
