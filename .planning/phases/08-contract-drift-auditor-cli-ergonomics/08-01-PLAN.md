<objective>
Add dependency-free CLI support for compact machine stdout, quiet mode, and opt-in fail policy to `contract-drift-auditor` while preserving packet artifacts as the source of truth.
</objective>

<context>
- Phase context: `.planning/phases/08-contract-drift-auditor-cli-ergonomics/08-CONTEXT.md`
- Research: `.planning/phases/08-contract-drift-auditor-cli-ergonomics/08-RESEARCH.md`
- Current CLI: `tools/contract-drift-auditor/cli.js`
- Runner return shape: `tools/contract-drift-auditor/index.js`
- CLI tests: `test/contract-drift-auditor/cli.test.js`
- Cross-repo CLI pattern: `tools/cross-repo-compatibility-checker/cli.js`
</context>

<must_haves>
- CLI-01: support stable machine stdout mode without replacing packet artifacts.
- CLI-02: support quiet mode and documented fail-on behavior.
- D-01: add `--format json`; unknown formats exit `2`.
- D-02: JSON stdout is compact and includes packet status, counts, generated artifacts, and output directory.
- D-03: default human stdout remains short and should include packet status.
- D-04: `--quiet` suppresses human success chatter but not errors, help, or explicit JSON output.
- D-05: `--fail-on blocked|human_review_required|never`, default `never`.
- D-06: do not change review packet schemas or generated artifacts.
- Do not add a CLI framework dependency.
- Preserve `--fix` and `--write` rejection.
</must_haves>

<threat_model>
- **T-08-01-01 Packet/source drift (high):** CLI stdout could recompute status or counts differently from `REVIEW-SUMMARY.json`.
  - **Mitigation:** build stdout from `runAudit` result and `packet.summary`.
  - **Verification:** tests compare JSON stdout status/counts to generated packet summary.
- **T-08-01-02 Breaking default shell behavior (high):** Existing users could get non-zero exit codes when drift is found.
  - **Mitigation:** default `--fail-on never`.
  - **Verification:** human-review fixture exits `0` without fail-on.
- **T-08-01-03 Quiet hides actionable errors (medium):** `--quiet` could suppress usage or safety errors.
  - **Mitigation:** apply quiet only to success stdout in human mode.
  - **Verification:** missing args and unsafe output still write stderr and return `2`.
- **T-08-01-04 Unsafe blocked-status test (medium):** Tests could weaken output isolation to create a blocked packet.
  - **Mitigation:** test blocked fail policy through a pure helper if real safe runner cannot produce `blocked`.
  - **Verification:** output isolation tests still prove target-local `--out` writes no artifacts.
</threat_model>
