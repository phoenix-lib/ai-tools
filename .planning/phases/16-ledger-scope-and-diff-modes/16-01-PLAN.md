# Plan 16-01: Ledger Scope Filtering and Source Categories

## Gate Resolution

- **Research Gate:** user selected research-first; evidence is in
  `16-RESEARCH.md`.
- **Dependency Gate:** depends on Phase 14 ledger schema discipline. Update
  schemas and generated-output tests with any new source category fields.
- **AI Tools Self-Use Gate:** applies later in `16-02` after scope and diff
  behavior are both implemented. This plan uses fixtures as the oracle.
- **New Tool Intake Gate:** skipped with reason. This plan extends the
  validated `project-context-ledger`; it does not introduce a new tool.
- **Cross-Repo Gates:** skipped with reason. No `ai-workspace-kit` mutation,
  dependency, or cross-repo request handling is in scope.

<objective>
Add explicit `project-context-ledger --scope current|planning|history|all`
support with `current` as the default, classify discovered sources
deterministically, and prove historical `.planning/phases/**` artifacts no
longer dominate default current-scope ledger findings.
</objective>

<must_haves>
- LEDGER-SCOPE-01: CLI supports `--scope current|planning|history|all`, with
  `current` as the default.
- LEDGER-SCOPE-03: current scope excludes historical `.planning/phases/**`
  artifacts by default.
- LEDGER-SCOPE-04: examples, placeholders, `n/a`, and generated packet
  artifacts are classified so they do not inflate current findings.
- D-01 through D-06: implement the four scope semantics exactly and preserve
  read-only output isolation.
- D-07 through D-10: add deterministic source category metadata and update
  schemas/tests when new fields are emitted.
- D-11 through D-14: history remains available through `history` and `all`;
  no safe-ignore or suppression semantics.
- D-15 through D-18: non-real references are classified conservatively.
- D-30 through D-32: only add narrow Phase 16 CLI flags; no Phase 17 shared CLI
  behavior.
</must_haves>

<threat_model>
- **T-16-01-01 Hidden history (high):** default current scope could make users
  think old evidence disappeared.
  - **Mitigation:** implement explicit `history` and `all`, document scope in
    manifest metadata, and test that history findings are present when asked.
  - **Verification:** discovery/integration tests compare current vs history
    or all outputs.
- **T-16-01-02 Schema drift (high):** source category fields could violate
  strict Phase 14 schemas.
  - **Mitigation:** update public schemas and schema-output tests in the same
    plan as generator changes.
  - **Verification:** generated ledger artifacts validate against updated
    schemas.
- **T-16-01-03 False negative references (high):** placeholder/example
  classification could hide real stale refs.
  - **Mitigation:** classifier is conservative; unknown cases remain real.
  - **Verification:** fixture covers real missing refs beside examples,
    placeholders, `n/a`, and generated packet dirs.
- **T-16-01-04 Scope leakage (medium):** historical phase docs could still be
  read before filtering and generate current findings.
  - **Mitigation:** filter source documents before markdown reference and
    documented-command extraction.
  - **Verification:** current-scope test asserts historical-only missing refs
    produce no default finding.
- **T-16-01-05 CLI scope creep (medium):** implementation could drift into the
  Phase 17 shared CLI contract.
  - **Mitigation:** only add `--scope`; no `--format`, `--quiet`, or
    `--fail-on`.
  - **Verification:** CLI tests cover only narrow scope parsing and existing
    mutating flag rejection.
</threat_model>
