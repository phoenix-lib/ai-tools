  `16-RESEARCH.md`.
- **Dependency Gate:** depends on `16-01` scope/category metadata. Do not build
  diff behavior before current/history filtering and schemas are stable.
- **AI Tools Self-Use Gate:** applies. Run `project-context-ledger` against
  this repository with default current scope and explicit history/all scope,

  scope, validation, compatibility, migration notes, and upstream impact.

  `ai-workspace-kit` or make AI Tools a runtime dependency there.

</must_haves>

<threat_model>
- **T-16-02-01 Silent no-diff fallback (high):** malformed or missing
  `--since-manifest` input could be ignored.
  - **Mitigation:** validate existence/JSON/schema enough before running; fail
    with exit code 2 or a blocked packet finding.
  - **Verification:** CLI tests cover missing, malformed, and valid manifest
    paths.
- **T-16-02-02 Semantic triage creep (high):** diff could rank business
  importance or decide changed facts are safe/unsafe.
  - **Mitigation:** output only mechanical categories and counts.
  - **Verification:** tests/docs assert no priority, safe-to-ignore,
    suppression, gate, merge, or roadmap decision fields.
- **T-16-02-03 Unstable diffs (high):** record comparison could include run
  timestamps or order-dependent JSON, making unchanged records appear changed.
  - **Mitigation:** hash canonical record objects after excluding volatile
    fields where necessary and document the comparison.
  - **Verification:** deterministic fixed-clock diff tests.
- **T-16-02-04 Artifact contract drift (medium):** new `LEDGER-DIFF.json`
  could be emitted but omitted from metadata, schemas, or docs.
  - **Mitigation:** update `shared/tool-metadata.js`, standards README,
    tool README, registry, and schema tests together.
  - **Verification:** schema-output and registry/release docs tests.
- **T-16-02-05 Self-use overclaiming (medium):** real repo counts could be
  treated as proof of semantic correctness.
  - **Mitigation:** self-use is evidence only; synthetic fixtures remain the
    oracle.
  - **Verification:** summary records both fixture tests and real output
    interpretation separately.
</threat_model>
