# Plan 13-02: Rollup CLI, Packet Rendering, Registry, Docs, and Self-Use Evidence

## Gate Resolution

- **Research Gate:** user selected Research first; evidence in `13-RESEARCH.md`.
- **Dependency Gate:** depends on `13-01` validation, normalization, and
  grouping modules. Do not start user-facing CLI/docs maturity work until the
  ingestion core has focused tests.
- **New Tool Intake and Placement Gate:** resolved in `13-CONTEXT.md`. Register
  rollup as an AI Tools-owned packet consumer at
  `tools/review-packet-rollup/`.
- **AI Tools Self-Use Gate:** applies. After the CLI is runnable, run rollup
  over at least two existing external self-use packet directories as evidence.
- **Project Changelog Gate:** applies. Record command surface, validation,
  self-use evidence, compatibility impact, breaking changes, and migration
  notes.
- **Cross-Repo Gates:** skipped with reason. This plan does not mutate
  `ai-workspace-kit`, copy upstream planning state, or add a runtime
  dependency.

<objective>
Wire the `review-packet-rollup --packets <dir...> --out <dir>` CLI, render the
standard review packet plus `PACKET-INDEX.json` and `ROLLUP-GROUPS.json`, update
metadata/docs/registry truthfully, validate generated outputs against schemas,
and record self-use evidence without making rollup a gate, runner, suppression
system, or semantic decision layer.
</objective>

<threat_model>
- **T-13-02-01 CLI scope creep (high):** implementation could pull in full
  Phase 17 CLI contract work.
  - **Mitigation:** only implement `--packets`, `--out`, `--help`, human stdout,
    and mutating/source-running flag rejection.
  - **Verification:** CLI tests reject `--format`, `--quiet`, and `--fail-on`
    unless deliberately implemented with focused tests and documented as
    non-breaking.
- **T-13-02-02 Output inside packet inputs (critical):** rollup could write
  generated artifacts into a source packet directory.
  - **Mitigation:** call `assertSafeOutputDirOutsideAll(packetDirs, outDir)`
    before loading or writing.
  - **Verification:** unsafe-output integration test and tree-hash checks for
    every input packet fixture.
- **T-13-02-03 Extra artifacts drift from summary schema (medium):**
  `REVIEW-SUMMARY.json.generated_artifacts` enum only permits the four standard
  packet artifacts.
  - **Mitigation:** keep summary `generated_artifacts` to
    `REQUIRED_PACKET_ARTIFACTS`; include `PACKET-INDEX.json` and
    `ROLLUP-GROUPS.json` in the tool manifest generated/requested files and
    docs.
  - **Verification:** schema-output tests validate the summary and assert extra
    artifacts are present on disk and in tool metadata, not summary enum.
- **T-13-02-04 Registry truth drift (medium):** registry could claim validated
  maturity before tests/docs/self-use pass.
  - **Mitigation:** update maturity only after focused tests, full suite, docs,
    and self-use evidence exist.
  - **Verification:** registry tests and `13-02-SUMMARY.md` record evidence.
- **T-13-02-05 Semantic decision language (medium):** docs or recommended
  actions could imply automatic suppression, merge approval, or gate approval.
  - **Mitigation:** use "evidence", "mechanical grouping", and "human review"
    wording; never use safe-ignore/disposition language.
  - **Verification:** docs review and tests assert forbidden terms are absent
    from generated groups.
</threat_model>
