<threat_model>
- **T-10-01-01 Semantic gate-review creep (high):** the seed could define a
  tool that decides whether gates should exist or be adopted.
  - **Mitigation:** README and seed state that `gates-scan` reports mechanical
    evidence only and final decisions remain assistant/human-owned.
  - **Verification:** docs test asserts evidence-only and non-decision wording.
- **T-10-01-02 Fixture gaps hide critical checks (high):** implementation could
  ship without proving the required failure cases.
  - **Mitigation:** create fixture contract tests that require all minimum
    fixture names and expected finding IDs before implementation.
  - **Verification:** `npm.cmd test -- test/gates-scan/fixture-contract.test.js`.
- **T-10-01-03 Registry maturity drift (medium):** registry could imply a
  runnable or validated tool before one exists.
  - **Mitigation:** keep maturity `planned` in this plan; only update
    `status_notes`, evidence refs, and non-goals if needed.
  - **Verification:** existing tool registry test still passes.
- **T-10-01-04 Kit dependency leak (medium):** fixture contract could require a
  sibling kit checkout.
  - **Mitigation:** fixtures are single-project AI Tools-like projects; kit
    files are referenced only as boundary evidence.
  - **Verification:** fixture contract test asserts no fixture path depends on
    `.external/ai-workspace-kit`.
</threat_model>
