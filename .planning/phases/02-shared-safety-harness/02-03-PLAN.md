# Plan 02-03: Add Integrated Safety Harness Tests

<objective>
Prove the Phase 2 safety harness end to end by combining output path guard,
secret policy, ignore/file walking, tree hashing, and target fixtures in
deterministic Node tests.
</objective>

<context>
- Depends on helper modules from plan 02-01.
- Depends on tree hash and fixtures from plan 02-02.
- This plan is still helper/test work only; it must not implement
  `contract-drift-auditor` behavior.
</context>

<must_haves>
- D-17: Tests include secret sentinels and prove those sentinels never appear in outputs.
- D-20: Generated packet directories are ignored by marker files.
- D-23: Walker output remains sorted relative `/` paths.
- D-24: Secret classification happens before any content read.
- D-26: Tests exercise all TEST-07 fixture scenarios.
- D-28: Mutation proof uses raw target tree hash, not audit ignore policy.
- D-30: No-mutation tests compare before/after tree hashes.
- D-31: Test outputs live outside target inputs and are cleaned.
- D-38: Upstream freshness was checked before planning.
- D-40: Use updated upstream framework-adapter lessons only to keep helpers generic; do not add framework detection.
- SAFE-01: Unsafe target-local output paths are rejected.
- SAFE-02: Secret-like files are path-only by default.
- SAFE-03: Ignore policy excludes generated packets and noisy dirs.
- SAFE-05: Fixture harness proves non-mutation.
- TEST-02: Target fixtures are not mutated by read-only commands.
- TEST-03: Secret sentinel strings never appear in output.
- TEST-07: Fixture coverage includes all required scenarios.
</must_haves>

<threat_model>
- **T-02-03-01 False read-only proof (high):** Tests might only check path guard and miss helper functions that mutate inputs.
  - **Mitigation:** Wrap representative read-only helper operations with before/after `treeHash` checks.
  - **Verification:** `test/shared/safety-harness.test.js` fails if any fixture input hash changes.
- **T-02-03-02 Sentinel leakage through output aggregation (critical):** Secret values could appear in JSON, Markdown, or thrown error text.
  - **Mitigation:** Build output text only from path-only evidence and walker paths, then scan all generated/returned text for sentinel strings.
  - **Verification:** `test/shared/safety-harness.test.js` asserts sentinel strings are absent from collected output text.
- **T-02-03-03 Generated packet evidence pollution (medium):** Existing generated packets inside target trees could be walked as current evidence.
  - **Mitigation:** Assert marker-based generated packet dirs are excluded by the file walker.
  - **Verification:** `test/shared/file-walker.test.js` and integration harness checks use the `generated-packet-inside-target` fixture.
</threat_model>
