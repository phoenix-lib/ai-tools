<threat_model>
- **T-07-03-01 Schema overreach (medium):** Checker could force ai-tools registry to become kit camelCase.
  - **Mitigation:** validate through `interop.field_name_mapping` and `interop.stage_aliases`.
  - **Verification:** registry test accepts explicit direct-compatibility false plus mapping.
- **T-07-03-02 Packet drift (high):** New checker packet could diverge from shared schema/artifact contract.
  - **Mitigation:** use shared renderer and schema validation tests.
  - **Verification:** generated checker packet validates against review packet schemas.
- **T-07-03-03 Hidden dependency creep (high):** CLI docs could imply it pulls, installs, or runs kit.
  - **Mitigation:** README and tests state read-only file inspection only.
  - **Verification:** docs tests/search plus no child process execution in implementation.
- **T-07-03-04 Self-use output pollution (medium):** Self-use run could write generated packet into either repo.
  - **Mitigation:** require external output path and record it as evidence.
  - **Verification:** command rejects repo-local output and successful self-use uses external output.
</threat_model>
