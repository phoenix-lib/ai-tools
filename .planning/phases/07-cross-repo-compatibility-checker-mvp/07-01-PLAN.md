<threat_model>
- **T-07-01-01 Hidden mutation (high):** The checker could write output inside one of the input repos.
  - **Mitigation:** Add or reuse output guard against both roots before packet write.
  - **Verification:** tests reject output inside `ai-tools` fixture and inside `ai-workspace-kit` fixture.
- **T-07-01-02 Brittle real-repo tests (medium):** Tests could depend on local sibling repo state.
  - **Mitigation:** Use small committed fixtures for normal tests.
  - **Verification:** fixture tests run without `C:\projects\ai-workspace-kit`.
- **T-07-01-03 Parser ambiguity (medium):** Broad text parsing could confuse prose with metadata.
  - **Mitigation:** parse only top-level `Field: value` lines for known fields.
  - **Verification:** protocol parser unit tests cover missing fields, repeated fields, and headings.
- **T-07-01-04 Registry creep (medium):** Metadata changes could become the future tool registry.
  - **Mitigation:** only add a checker tool-name constant if needed.
  - **Verification:** no owner/maturity/activation registry data added to `shared/tool-metadata.js`.
</threat_model>
