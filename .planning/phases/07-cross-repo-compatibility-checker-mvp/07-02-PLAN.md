# Plan 07-02: Protocol Thread, Counterpart, Origin, Decision, and Path Checks

## Gate Resolution

- **Research Gate:** user selected `Research first`; evidence in `07-RESEARCH.md`.
- **Upstream Freshness Gate:** passed for planning. Use `AI-WORKSPACE-CONTRACT.json` and protocol version evidence when present, but do not require them from older kit checkouts.
- **Cross-Repo Incoming Review Gate:** passed. Checks validate request compatibility; they do not decide whether requests should be accepted.
- **Cross-Repo Outgoing Need Gate:** no outgoing request needed. This plan implements AI Tools-owned external validation.
- **AI Tools Self-Use Gate:** skip until `07-03`; after packet rendering exists, run the checker against real local repos.
- **Project Changelog Gate:** applies because protocol validation capability changes tool behavior.

<objective>
Implement deterministic compatibility checks for request metadata, semantic threads, counterpart links, manual-transfer decisions, and portable paths across the two repository inputs.
</objective>

<must_haves>
- XREPO-VALIDATOR-01: checker validates protocol version, canonical request IDs, Thread ID, Origin, Mirror required, counterpart IDs, and repo-qualified counterpart paths.
- XREPO-VALIDATOR-01: checker requires decisions for manual-transfer requests with `Mirror required: false`.
- D-05: deterministic field extraction only.
- D-06: group mirrored requests by `Thread ID`.
- D-07: require local decision evidence for manual-transfer requests.
- D-08: report machine-local absolute paths.
- D-12: all checks covered by small paired fixtures.
- D-13: findings are evidence only and do not accept/reject requests.
</must_haves>

<threat_model>
- **T-07-02-01 False duplicate requests (high):** Different canonical IDs could be treated as separate incompatible requests.
  - **Mitigation:** group semantic compatibility by `Thread ID`.
  - **Verification:** fixture with different canonical IDs and same thread passes.
- **T-07-02-02 Unreviewed manual transfer (high):** A manually transferred request could bypass decision evidence.
  - **Mitigation:** require a decision artifact when `Origin: manual-transfer` and `Mirror required: false`.
  - **Verification:** missing-decision fixture reports `human_review_required`.
- **T-07-02-03 Broken counterpart silently passes (high):** Counterpart path may point nowhere or to the wrong thread.
  - **Mitigation:** resolve repo-qualified counterpart paths and compare thread metadata.
  - **Verification:** broken path and wrong-thread fixtures produce findings.
- **T-07-02-04 Local path leakage (medium):** Absolute paths could make protocol artifacts non-portable.
  - **Mitigation:** scan protocol metadata/content for machine-local paths and emit findings.
  - **Verification:** absolute-path fixture produces a finding with path-only evidence.
</threat_model>
