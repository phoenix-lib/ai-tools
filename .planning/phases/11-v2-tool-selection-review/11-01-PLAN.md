- **Upstream Freshness Gate:** resolved in `11-UPSTREAM-UPDATE-REVIEW.md`.
  Embedded `ai-workspace-kit` is current at `b5903a4`; the update reinforces
  structured impact review and evidence-only external tool boundaries.
- **New Tool Intake and Placement Gate:** applies. Promote exactly one tool,
  `project-context-ledger`, to `planned` with owner, destination, outputs,
  use gate, activation stages, non-goals, and fixture seed. Do not implement it.

  roadmap, changelog, or selection artifacts. Run relevant validated tools as
  evidence only.

  implementation is needed.

  registry truth, and next-tool selection evidence.

<threat_model>
- **T-11-01-01 Premature implementation (high):** Phase 11 could add runnable
  ledger code or package metadata.
  - **Mitigation:** selection artifacts and registry metadata only; no
    `tools/project-context-ledger/cli.js`, package bin, package script, or
    generated ledger output.
  - **Verification:** focused planning test asserts package metadata does not
    expose `project-context-ledger`.
- **T-11-01-02 Multiple-tool expansion (high):** the phase could promote
  several appealing seeds at once.
  - **Mitigation:** selection test asserts exactly one formerly deferred v2
    seed is `planned`, and the review artifact states non-selected reasons.
  - **Verification:** `npm.cmd test -- test/planning/tool-selection.test.js`.
- **T-11-01-03 Requirement truth drift (high):** `LEDGER-01` could be marked
  complete even though no scanner exists.
  - **Mitigation:** add a future roadmap phase for the ledger MVP or otherwise
    mark `LEDGER-01` as planned, not complete.
  - **Verification:** roadmap/requirements assertions in the planning test.
- **T-11-01-04 Kit boundary creep (medium):** selection could copy upstream
  dependency lifecycle or gate behavior into AI Tools.
  - **Mitigation:** selection review cites upstream as boundary evidence only
    and keeps ledger ownership in `ai-tools`.
  - **Verification:** review artifact and registry non-goals mention no kit
    dependency or automatic execution.
- **T-11-01-05 Self-use overreach (medium):** validated tools could be treated
  as automatic decisions.
  - **Mitigation:** self-use output is recorded as evidence only in summary or
    verification artifacts.
  - **Verification:** changelog and summary wording preserve evidence-only
    semantics.
</threat_model>
