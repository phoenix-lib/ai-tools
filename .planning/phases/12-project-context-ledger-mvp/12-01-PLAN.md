<threat_model>
- **T-12-01-01 Target mutation (critical):** the ledger could write inside or
  otherwise alter the scanned project.
  - **Mitigation:** require `--out`, call `assertSafeOutputDir` before any
    writes, write only to external output, and use fixture tree hashes.
  - **Verification:** unsafe output test and no-mutation integration test.
- **T-12-01-02 Secret leakage (critical):** secret-like file contents could
  appear in evidence, hashes, facts, or Markdown.
  - **Mitigation:** use `isSecretLikePath` / `secretEvidenceRef`; do not read
    secret-like contents; represent secrets as path-only evidence.
  - **Verification:** sentinel secret fixture must not appear in any output.
- **T-12-01-03 Generated packet recursion (high):** prior review packets inside
  a target could be scanned as source facts and pollute the ledger.
  - **Mitigation:** use `walkProjectFiles` and `shouldIgnoreDir` generated
    packet detection.
  - **Verification:** fixture with target-local generated packet directory is
    ignored.
- **T-12-01-04 Context hallucination (high):** the ledger could invent project
  facts or imply source reading is unnecessary.
  - **Mitigation:** facts require evidence refs and confidence; unknown/stale
    values are explicit; docs state ledger output is evidence only.
  - **Verification:** tests assert missing evidence becomes unknown/stale or a
    finding, not a verified invented fact.
- **T-12-01-05 Packet/schema drift (high):** review summary counts, Markdown,
  and JSON could diverge.
  - **Mitigation:** use shared packet renderer and existing schemas.
  - **Verification:** schema-output and deterministic packet tests.
- **T-12-01-06 Dependency lifecycle creep (medium):** package metadata scanning
  could become vulnerability/update/abandonment advice.
  - **Mitigation:** record commands/bins/scripts only; exclude dependency
    health and update advice from checks and docs.
  - **Verification:** README and tests confirm no dependency scanner or package
    runner behavior.
- **T-12-01-07 Registry truth drift (medium):** registry could claim validated
  status before the CLI, tests, docs, and self-use evidence exist.
  - **Mitigation:** update maturity only after validation; record evidence refs.
  - **Verification:** planning registry tests and final self-use summary.
</threat_model>
