# Plan 01-03: Add Canonical JSON Guidance, Examples, and Schema Validation Tests

<objective>
Prove the review packet standard by adding canonical JSON guidance, valid example packets, and Node tests for schema validation, deterministic JSON serialization, and summary count consistency.
</objective>

<context>
- Depends on complete schemas from plan 01-02.
- Canonical JSON reference: `.external/ai-workspace-kit/scripts/lib/canonical-json.js`.
- Shared summary reference: `.external/ai-workspace-kit/scripts/lib/review.js`.
- Phase 1 should add only the minimum test harness needed to prove the standard.
</context>

<must_haves>
- D-01: Example packets include required artifacts: `REVIEW-SUMMARY.json`, `EVIDENCE.json`, `FINDINGS.md`, and `RECOMMENDED-ACTIONS.md`.
- D-03: Markdown examples clearly project JSON data instead of becoming a separate source of truth.
- D-04: `EVIDENCE.json` examples are machine evidence indexes and contain no long copied source content.
- D-18: Tests prove summary counts match findings across JSON and Markdown example projections.
- D-20: Canonical JSON is recursively sorted object keys plus a trailing newline.
- D-21: Secret-like evidence examples use `path_only: true` and do not include content hashes.
- D-24: Examples mark unknown or stale facts explicitly when present.
- D-25: Examples do not include evidence snippets.
- TEST-01: Tests prove deterministic generated review output.
- TEST-06: Tests prove generated packet JSON validates against schemas.
</must_haves>

<task id="T2" type="create">
<files>
- `standards/review-packet/CANONICAL-JSON.md`
</files>
<action>
Document canonical JSON rules:
- Recursively sort object keys.
- Preserve array order.
- Serialize with two-space indentation.
- End JSON files with exactly one trailing newline.
- Hash canonical JSON text when hashing structured packet data.
- For secret-like path-only evidence, do not hash secret file contents.
- Future implementations should use one shared writer rather than ad hoc `JSON.stringify`.
</action>
<verify>
PowerShell:
`Select-String -Path standards/review-packet/CANONICAL-JSON.md -Pattern 'recursively sort','trailing newline','Preserve array order','path-only'`
</verify>
<acceptance_criteria>
Canonical JSON guidance is specific enough for Phase 2 to implement a shared helper and for Phase 1 tests to assert determinism.
</acceptance_criteria>
</task>
