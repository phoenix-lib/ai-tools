# Plan 05-01: Write Release Docs and Validate Required Guidance

## Gate Resolution

- **Research Gate:** user selected `Research first`; evidence in `05-RESEARCH.md`.
- **Upstream Freshness Gate:** passed before planning; local and remote
  `ai-workspace-kit` commits match `2079ab9626e9f9ed256512091f9c5ea473582885`.
- **Cross-Repo Incoming Review Gate:** existing incoming gate-review/changelog
  request is already planned; this plan documents the accepted process only.
- **Cross-Repo Outgoing Need Gate:** not applicable; this plan does not require
  new kit-owned capability.
- **AI Tools Self-Use Gate:** applies later in plan `05-02`; this plan creates
  the docs that will interpret self-use evidence.
- **New Tool Intake Gate:** no new v1 tool is promoted.
- **Project Changelog Gate:** update `CHANGELOG.md` because this plan changes
  release docs and downstream planning evidence.

<objective>
Create the first-release documentation surface for AI Tools and add mechanical
docs validation so release-critical guidance cannot disappear silently.
</objective>

<context>
- Phase context: `.planning/phases/05-integration-and-release-hardening/05-CONTEXT.md`
- Research: `.planning/phases/05-integration-and-release-hardening/05-RESEARCH.md`
- Packet standard: `standards/review-packet/README.md`
- Current auditor docs: `tools/contract-drift-auditor/README.md`
- Existing docs validation pattern: `test/planning/cross-repo-protocol.test.js`
- Package state: `package.json` is `private: true`; docs must use repo-local commands.
</context>

<must_haves>
- DOC-01: documentation explains when to use `contract-drift-auditor` and when not to use it.
- DOC-02: documentation explains that `ai-workspace-kit` remains the adoption/bootstrap contract tool and external auditors are optional.
- DOC-03: documentation explains secret handling, output isolation, evidence refs, severity, confidence, and unknown/stale facts.
- DOC-04: first release definition of done is documented and references the shared packet schema plus one working external auditor.
- Root `README.md` exists and is the first consumer entrypoint.
- Documentation must not imply npm publication or automatic cross-repo execution.
- `ai-workspace-kit` integration stays optional, packet-based, and dependency-free.
- Markdown docs are human guidance; `REVIEW-SUMMARY.json` remains the machine source of truth for packet output.
</must_haves>

<threat_model>
- **T-05-01-01 Misleading install guidance (medium):** Docs could imply a published package or global CLI.
  - **Mitigation:** Use repo-local commands only.
  - **Verification:** docs validation checks for local command examples and absence of publish assumptions.
- **T-05-01-02 Boundary creep (high):** Docs could imply `ai-workspace-kit` runs AI Tools automatically or depends on it.
  - **Mitigation:** Add explicit optional/evidence-only boundary language.
  - **Verification:** docs validation checks optional boundary phrases and no automatic dependency language.
- **T-05-01-03 Release checklist becomes prose-only (medium):** Release readiness could be non-checkable.
  - **Mitigation:** Use checklist sections with concrete evidence commands/artifacts.
  - **Verification:** docs validation checks required release readiness terms.
</threat_model>
