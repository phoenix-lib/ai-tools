  `09-RESEARCH.md`.

  `upstream-freshness` gate with update-impact fields after reviewing kit
  commit `485da62`.

  evidence after changing registry, gate docs, or `AGENTS.md`.

  gate docs, upstream update-impact, validation, and compatibility impact.

<threat_model>
- **T-09-02-01 Over-slimming safety rules (high):** moving gate policy out of
  `AGENTS.md` could hide critical preflight rules.
  - **Mitigation:** keep hard safety rules, source layers, discuss-mode
    preflight, and canonical links in root.
  - **Verification:** focused tests assert retained root phrases and links.
- **T-09-02-02 Duplicate upstream gate ids (medium):** adding Kit Update
  Self-Check as a separate gate could split enforcement.
  - **Mitigation:** update existing `upstream-freshness` required fields and
    docs; use Kit Update Self-Check as prose name only.
  - **Verification:** tests assert a single `upstream-freshness` id and the new
    update-impact fields.
- **T-09-02-03 Docs drift from registry (medium):** docs could name tools or
  self-use stages that the registry does not contain.
  - **Mitigation:** docs link to `tools/registry.json` instead of duplicating
    every entry.
  - **Verification:** docs tests assert links and required wording.
- **T-09-02-04 Self-use evidence treated as authority (medium):** tool output
  could be interpreted as an automatic decision.
  - **Mitigation:** gate docs repeat evidence-only boundary.
  - **Verification:** self-use packet summaries are recorded as evidence with
    assistant interpretation and residual risk.
</threat_model>
