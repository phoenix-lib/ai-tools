# Plan 09-01: Tool Registry Schema, Data, and Validation

## Gate Resolution

- **Research Gate:** user selected `Research first`; evidence in
  `09-RESEARCH.md`.
- **Upstream Freshness Gate:** resolved. `.external/ai-workspace-kit`
  fast-forwarded from `7bc432c` to `485da62`; Phase 9 adopts the
  machine-readable contract/index pattern and consumer-practice impact idea
  without copying kit doctor/smoke tooling.
- **Cross-Repo Incoming Review Gate:** resolved. Existing incoming requests
  support clearer registry and gate routing; they do not require a new Phase 9
  tool.
- **Cross-Repo Outgoing Need Gate:** skipped with reason. The registry is
  AI Tools-owned governance and needs no kit-owned capability.
- **AI Tools Self-Use Gate:** resolved for planning with the cross-repo checker;
  execution self-use for `contract-drift-auditor` belongs in `09-02` after
  docs and `AGENTS.md` change.
- **New Tool Intake Gate:** resolved. This plan adds a capability catalog, not
  a new runnable external tool. `gates-scan` remains planned Phase 10.
- **Project Changelog Gate:** applies in `09-02` because registry and workflow
  gate docs change downstream planning evidence.

Create the machine-readable AI Tools capability registry and validation tests
so self-use, new-tool intake, and future `gates-scan` planning can rely on a
deterministic catalog instead of scattered prose.
</objective>

<must_haves>
- REG-01: registry records owner, destination, maturity, self-use stage,
  required outputs, activation gate, and non-goals.
- D-01: create `tools/registry.json` and `tools/registry.schema.json`.
- D-02: keep entries explicit and schema-validated.
- D-03: separate `maturity` from `self_use.required`.
- D-04: include implemented tools, planned near-term tools, deferred seeds, and
  kit-owned boundary entries.

  prevent duplication.
- D-07: register `gates-scan` as planned/evidence-only, but do not implement
  it.
- Do not create a runtime registry loader unless tests need a tiny helper.
- Do not auto-run tools from registry data.
</must_haves>

<threat_model>
- **T-09-01-01 Vague registry entries (high):** future agents could add
  unbounded tool ideas with no owner or non-goals.
  - **Mitigation:** strict schema requiring owner, destination, maturity,
    activation stage, outputs, use gate, self-use, non-goals, and evidence refs.
  - **Verification:** registry validation test rejects missing required fields
    through schema validation.
- **T-09-01-02 Package/bin drift (medium):** implemented CLIs could be missing
  from the registry.
  - **Mitigation:** test every `package.json` `bin` key has a registry entry
    with runnable metadata.
  - **Verification:** `test/planning/tool-registry.test.js`.
- **T-09-01-03 Phase 10 leakage (medium):** registry could imply `gates-scan`
  already exists.
  - **Mitigation:** maturity `planned`, destination future path, non-goal
    "no Phase 9 CLI/check implementation".
  - **Verification:** test asserts `gates-scan` is not in package `bin`.
- **T-09-01-04 Kit boundary drift (medium):** registry could classify
  kit-owned adoption/gate behavior as AI Tools-owned.
  - **Mitigation:** include a boundary entry for `ai-workspace-kit-internal-gates`
    with owner `ai-workspace-kit`.
  - **Verification:** tests assert boundary entry owner and non-goals.
</threat_model>

<action>
Add a JSON schema for the tool registry. Require:

- top-level `schema_version` and `tools`;
- each tool `id`, `name`, `owner`, `destination`, `maturity`,
  `activation_stage`, `expected_outputs`, `use_gate`, `self_use`,
  `non_goals`, and `evidence_refs`;
- constrained owner values: `ai-tools`, `ai-workspace-kit`, `shared-boundary`;
- constrained maturity values: `validated`, `implemented`, `planned`,
  `seed-only`, `deferred`;
- self-use object with `required`, `stages`, and `reason`;
- non-empty arrays for activation stages, non-goals, and evidence refs.

Then create `tools/registry.json` with initial entries for:

- `contract-drift-auditor`;
- `cross-repo-compatibility-checker`;
- `tool-registry`;
- `gates-scan`;
- `ai-workspace-kit-internal-gates`;
- all deferred seed directories currently listed in `tools/README.md`.
</action>

</verify>
