<threat_model>
- **T-04-01-01 Packet projection drift (high):** Markdown, JSON, evidence, actions, or CLI output could report different counts/statuses.
  - **Mitigation:** Add `shared/review-packet-renderer.js` and require all artifacts to render from one packet model.
  - **Verification:** `test/shared/review-packet-renderer.test.js` checks generated artifacts, counts, and canonical JSON.
- **T-04-01-02 Unsafe output write (high):** CLI could create reports inside the audited target.
  - **Mitigation:** Call `assertSafeOutputDir(project, out)` before creating output directories.
  - **Verification:** `test/contract-drift-auditor/cli.test.js` covers target-local `--out` rejection.
- **T-04-01-03 Evidence pollution (medium):** Discovery could treat generated packets or nested kit checkouts as current target evidence.
  - **Mitigation:** Use existing `walkProjectFiles` default ignore policy.
  - **Verification:** discovery tests include generated packet fixture and mature fixture.
</threat_model>
