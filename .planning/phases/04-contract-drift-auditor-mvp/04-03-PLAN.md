# Plan 04-03: Emit Review Packets and Prove Auditor Safety

## Gate Resolution

- **Research Gate:** uses `04-RESEARCH.md`.
- **Upstream Freshness Gate:** current kit commit and planning impact recorded in `.planning/research/AI-WORKSPACE-KIT-UPSTREAM-REVIEW.md`.
- **Project Changelog Gate:** this plan updates `CHANGELOG.md` after the auditor MVP is complete.
- **AI Tools Self-Use Gate:** after this plan, the auditor may become experimental/validated depending on verification results; record status in changelog or summary.

<objective>
Integrate discovery, drift checks, and the shared renderer into a complete
read-only auditor run that emits schema-valid review packet artifacts and proves
deterministic, non-mutating, secret-safe behavior across target fixtures.
</objective>

<context>
- Phase context: `.planning/phases/04-contract-drift-auditor-mvp/04-CONTEXT.md`
- Research: `.planning/phases/04-contract-drift-auditor-mvp/04-RESEARCH.md`
- Validation: `.planning/phases/04-contract-drift-auditor-mvp/04-VALIDATION.md`
- Renderer from plan 04-01: `shared/review-packet-renderer.js`
- Drift checks from plan 04-02: `tools/contract-drift-auditor/checks.js`
- Existing schema tests: `test/review-packet/schema-validation.test.js`
</context>

<must_haves>
- D-01: runnable review-only CLI named `contract-drift-auditor`.
- D-02: reject target-local output paths.
- D-03: no installs, target command execution, target mutation, copied `.planning`, or auto-fix.
- D-04: self-audit output is isolated from target.
- D-05: checks are deterministic and evidence-backed.
- D-11: generated packets inside target are ignored as current evidence.
- D-12: shared renderer is part of Phase 4.
- D-13: all artifacts and CLI status come from one packet model.
- D-14: canonical JSON output.
- D-15: findings cite evidence refs and recommended action refs.
- D-16: evidence paths are relative normalized `/`.
- D-17: secret-like files are path-only evidence.
- D-18: evidence stays narrow.
- D-19: incoming kit request accepted as Phase 4 scope.
- D-20: kit is optional consumer, not dependency.
- D-21: no kit-owned adoption/bootstrap duplication.
- D-22: cross-repo validator stays deferred v2.
- D-23: reuse existing target fixtures.
- D-24: expected outputs outside fixture `input/`.
- D-25: tests prove missing command, stale source-layer, schema-valid packets, deterministic output, non-mutation, output isolation, and secret safety.
</must_haves>

<threat_model>
- **T-04-03-01 Schema-invalid packet (high):** Runner could emit artifacts consumers cannot parse.
  - **Mitigation:** Validate generated summaries/findings/evidence/actions against existing schemas.
  - **Verification:** `schema-output.test.js` validates generated packet artifacts.
- **T-04-03-02 Target mutation (critical):** Auditor could write reports into target or mutate fixtures.
  - **Mitigation:** output path guard plus before/after `treeHash` around fixture input.
  - **Verification:** integration tests assert unchanged tree hash.
- **T-04-03-03 Secret sentinel leak (critical):** Secret-like fixture contents could appear in packet output.
  - **Mitigation:** path-only secret policy and artifact-wide sentinel search.
  - **Verification:** integration tests search all generated artifacts for sentinel strings.
- **T-04-03-04 Cross-repo dependency creep (medium):** Docs could imply kit runs AI Tools automatically.
  - **Mitigation:** README and changelog state optional, review-only, packet-based integration.
  - **Verification:** docs contain no auto-run/dependency language.
</threat_model>
