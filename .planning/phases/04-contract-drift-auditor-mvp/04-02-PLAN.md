<objective>

<must_haves>
- D-05: MVP checks are deterministic and evidence-backed.
- D-06: check referenced files/source layers including missing local paths.
- D-07: check referenced commands against package scripts and documented command evidence; do not execute commands.
- D-08: check permissions/workflow guidance referencing absent tools or package managers.
- D-09: check referenced skills for existence and readable `SKILL.md`.
- D-10: mark insufficient profile/source facts as `unknown`, `stale`, or `unresolved`; do not invent facts.
- D-11: ignore generated AI Tools and kit packets as current evidence by default.
- D-16: evidence paths are relative normalized `/` paths.
- D-17: secret-like files are path-only evidence.
- D-18: evidence stays narrow.
- D-23: reuse existing target fixtures.
- D-24: expected outputs stay outside fixture `input/` trees.
- D-25: tests prove missing command, stale/missing source layers, schema-valid packet output, deterministic output, non-mutation, output isolation, and secret safety.
</must_haves>

<threat_model>
- **T-04-02-01 False drift from naive path parsing (medium):** Parser may treat prose tokens as paths.
  - **Mitigation:** Use conservative path-looking patterns and only report missing paths when the reference is clearly file-like.
  - **Verification:** `checks.test.js` covers clean and mature fixtures with no false high-severity drift.
- **T-04-02-02 Command execution side effect (high):** Auditor could run target commands while verifying commands.
  - **Mitigation:** Read package scripts only; never spawn target commands.
  - **Verification:** tests assert missing-command fixture reports `drift.command.missing` without command execution.
- **T-04-02-03 Skill/permission boundary confusion (medium):** Auditor could grant or deny permissions instead of reporting evidence.
  - **Mitigation:** Findings recommend manual contract updates; do not mutate or approve permissions.
  - **Verification:** findings have recommended actions, not applied changes.
</threat_model>
